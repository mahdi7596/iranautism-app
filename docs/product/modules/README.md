# Module Notes

This directory stores implementation-facing notes for individual modules once design or development starts.

Before creating or updating a module file:

1. Read `docs/decisions/backend-architecture.md`.
2. Check the module status and open questions in `docs/product/module-registry.md`.
3. Record meaningful decisions in `docs/decisions/change-log.md`.
4. Update root `AGENTS.md` only when a short operational status pointer changes.

Use the per-module template in `docs/decisions/backend-architecture.md`.
